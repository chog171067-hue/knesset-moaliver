# מדריך פריסה - תיקון האזור האישי

## מה השתנה (תקציר)

| קובץ | סטטוס |
|---|---|
| `netlify/functions/lib/community.js` | **חדש** - מקור אמת יחיד לרשימת חברי הקהילה (מהגיליון) |
| `netlify/functions/lib/identity-admin.js` | **חדש** - עטיפה ל-Identity Admin API (רשימת משתמשים, עדכון) |
| `netlify/functions/verify-tz.js` | **מוחלף** - בודק זכאות אמיתית + האם קיים חשבון |
| `netlify/functions/identity-signup.js` | **מוחלף** - מאמת ת"ז בפועל בהרשמת מייל+סיסמה, דוחה כפילויות |
| `netlify/functions/bind-tz.js` | **חדש** - מקשר ת"ז לחשבון אחרי כניסה/הרשמה דרך Google |
| `netlify/functions/get-donor-data-by-tz.js` | **מוחלף** - **דורש אימות JWT**, לא מקבל tz/phone מהלקוח יותר |
| `personal.html` | **מוחלף** - זרימת התחברות/הרשמה נפרדות + קישור Google בטוח |

⚠️ שים לב: הקובץ הישן שנקרא `get-donor-data.js` **חייב להימחק** מהריפו (או לפחות להפסיק לשמש), אחרת יהיו לך שני קבצים כפולים. הקובץ החדש חייב להיקרא בדיוק `get-donor-data-by-tz.js`.

## שלב 1: פרסום גיליון ה-Google Sheets כ-CSV

בגיליון עצמו:
1. עמודה A: `תז`, עמודה B: `טלפון` (שורה ראשונה = כותרות, בדיוק כך).
2. ודא שמספרי הטלפון בגיליון נקיים ממקפים/רווחים, או שהם יתעדכנו כך אוטומטית ע"י הקוד.
3. תפריט **קובץ ← שיתוף ← פרסום לאינטרנט (Publish to web)**.
4. בחר את הגיליון הספציפי (לא "כל המסמך"), ובחר פורמט **CSV**.
5. לחץ פרסום, והעתק את הקישור שמתקבל.

⚠️ **הערת פרטיות**: קישור זה הופך את תוכן הגיליון לנגיש לכל מי שמחזיק בקישור. ודא שהגיליון הזה מכיל **רק** תז+טלפון, ולא סכומי תרומות/פרטים רגישים נוספים.

## שלב 2: משתני סביבה בנטליפיי

Site settings → Environment variables:

| שם המשתנה | ערך |
|---|---|
| `SHEET_CSV_URL` | הקישור מסעיף 1 |
| `NETLIFY_IDENTITY_ADMIN_TOKEN` | Personal Access Token (User settings → Applications → New access token) |
| `SITE_ID` | כבר אמור להיות מוגדר אצלך (נבדק בקובץ הישן) |
| `NEDARIM_API_PASSWORD` / `NEDARIM_API_PASSWORD_2` | כבר קיימים אצלך |

## שלב 3: הפעלת identity-signup כ-Event Function

ודא בהגדרות Identity באתר (Identity → Settings and usage → Emails/External providers/Registration) שה"Registration" מוגדר כ"Invite only" **לא** נדרש - הוא צריך להיות פתוח (Open), כי הבקרה על מי רשאי נעשית ע"י `identity-signup.js` ולא ע"י Netlify עצמו.

## שלב 4: תלות (axios)

כל הקבצים החדשים משתמשים ב-`axios`, בדיוק כמו הקובץ הישן `verify-tz.js` שכבר השתמש בו - כך שסביר שהתלות כבר מותקנת. אם לא: הוסף `axios` ל-`package.json` שבתיקיית `netlify/functions` (או השורש, בהתאם למבנה שלך).

## שלב 5: בדיקה מסודרת אחרי פריסה

1. הזן ת"ז **שלא** קיימת בגיליון → צריך לקבל הודעת סירוב, בלי אפשרות המשך.
2. הזן ת"ז זכאית **חדשה** → אמור להיפתח טופס **הרשמה** (לא התחברות), עם מייל+סיסמה או Google.
3. השלם הרשמה במייל+סיסמה, אשר את המייל, התחבר → אמור לראות נתונים.
4. נסה להירשם **שוב** עם אותה ת"ז ומייל אחר → אמור להידחות עם הודעה שהת"ז כבר משויכת.
5. נסה "המשך עם Google" עם ת"ז זכאית חדשה → אחרי החזרה מגוגל, המערכת אמורה לקשר אוטומטית ולהציג נתונים.
6. נסה לפתוח את ה-Developer Console ולשלוח בקשה ישירה ל-`/.netlify/functions/get-donor-data-by-tz` בלי Authorization header → אמור לקבל `401`.
